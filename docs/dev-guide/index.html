[用户手册] Setup & Operation Guide

面向最终使用者：如何在 ChatGPT 中配置并使用 github-connector。

01

部署前置条件

  • 一个 Vercel 账号，已导入本仓库并成功构建（vercel.json 已配置 /mcp 路由重写）
  • 一个自定义域名（可选，但推荐 —— 默认 *.vercel.app 生产域名可能受 Deployment Protection 影响）
  • 一个 GitHub OAuth App（github.com → Settings → Developer settings → OAuth Apps）—— 记下 client_id，生成 client_secret；回调地址填 <部署域名>/callback
  • 一个 Upstash Redis 实例（通过 Vercel Marketplace 集成添加，自动注入 KV_REST_API_URL / KV_REST_API_TOKEN）
02

在 ChatGPT 中添加 Connector

  1. ChatGPT → 设置 → Connectors → 开启 Developer Mode（如未开启）
  2. 点击「Create」/「新建 App」
  3. Server URL 填入你的部署地址 + /mcp，例如：
    https://github-connector.example.com/mcp
  4. Authentication 下拉选择 OAuth（唯一支持的模式；ChatGPT 通过服务端的 .well-known 元数据自动发现授权端点，无需手动填写）
  5. Client registration 选 Dynamic Client Registration (DCR)（服务端已实现 POST /register，无需手动创建 client）
  6. 勾选「I understand and want to continue」（OpenAI 未审核自定义 MCP server 的风险提示）
  7. 点击 Create，随后会跳转到 GitHub 登录/授权页 —— 每个使用者用自己的 GitHub 账号登录，服务端不代持共享密钥

05

已知限制 & 安全须知

  • 当前：多用户 GitHub OAuth 已上线 —— 每个 ChatGPT 使用者用自己的 GitHub 账号登录授权，服务端按 githubUserId 隔离每个人的 token，不再有共享身份
  • 当前：旧版单一共享 GITHUB_TOKEN / CONNECTOR_SECRET / ALLOW_NO_AUTH 方案已完全移除，无向后兼容 —— 见本页图示
  • Classic GitHub OAuth App 的 token 不会过期、也没有 refresh_token；MCP 层面的 access token 有 24 小时有效期，由 ChatGPT 自动静默刷新（用户无感知）
  • 授权粒度为 repo（读写公开 + 私有仓库）—— 相比旧版 Fine-grained PAT 更粗，这是 Classic OAuth App 不支持细粒度 scope 的已知取舍
  • 不支持删除文件 / 删除分支等破坏性操作 —— 这是有意为之的边界，不在当前计划范围内
